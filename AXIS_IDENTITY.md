# AXIS Identity

Last updated: 2026-07-02

## Purpose
AXIS is a private personal operating system for one user.
It exists to reduce friction, preserve context, and turn daily life into a stable, reviewable system.

## Owner profile
- Single-user system
- Primary location/timezone context: Alexandria, Egypt
- Main devices: phone + PC
- Uses Telegram, iPhone Shortcuts, browser dashboard, and coding agents
- Values privacy and direct control over tools

## Core life domains
1. Fitness
2. Sleep
3. Nutrition
4. Library / reading
5. Design / creative work
6. Finance
7. Music
8. Fast capture / clipboard / notes

## Design taste
- Minimal, clean, premium
- Apple-software clarity
- Dune-like warm futurism
- Orange / sand / muted neutral accents
- Less glow, less gamer UI, less cyberpunk noise
- Strong visual hierarchy, generous spacing, calm surfaces

## Behavioral preferences
- Concise over verbose
- Useful over clever
- Stable over flashy
- Modular over tangled
- Real sync over fake UI state
- Quiet background automation over noisy confirmations

## What AXIS should optimize for
- Cross-device continuity
- Low-friction capture
- Clear current-state visibility
- Fast logging
- Long-term memory
- Server-backed truth where it matters

## Non-goals
- Not a social app
- Not a generic SaaS dashboard
- Not a bloated productivity toy
- Not heavy enterprise workflow software
- Not a place for fake metrics or decorative features with no operational value

## Core philosophy
- If it matters, it should be tracked cleanly
- If it is tracked, it should sync reliably
- If it syncs, it should not interrupt the user
- If it looks futuristic, it should still feel calm and readable

## Agent interpretation rules
Any agent working on AXIS should assume:
- This is a personal OS, not a demo project
- Working features must be preserved unless explicitly replaced
- Sensitive context should be handled carefully
- Private identity/context files may exist and should not be carelessly exposed
